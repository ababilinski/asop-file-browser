# Security

Please do not report a security vulnerability in a public issue.

Use **Security → Report a vulnerability** on GitHub. This opens a private
discussion with the maintainer and keeps the report and any proof of concept
out of the public issue tracker.

Include the affected version, the steps needed to reproduce the issue, and the
impact you observed. Please do not include passwords, private keys, personal
files, or other sensitive data that is not needed to understand the report.
